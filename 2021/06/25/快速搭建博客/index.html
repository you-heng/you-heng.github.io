部署

hexo博客可以直接放在github上面，不需要自己买服务器，白嫖github

_config.yml

在根目录下的_config.yml中设置

github

检查电脑是否有sshkey的设置

如果没有，会提示No such file or directory需要自己生成key

会有三个回车，回车的意思是设置提交密码，可以设置也可以不设置，不设置就直接回车

将生成的公钥id_rsa.pub添加到github账户中

自动化部署

github有一个github Action的功能，可以自动化部署，我们这里就用它来将博客的源码发布到github上，让它自动部署博客

准备

需要准备两个github仓库，一个用来放博客所有源文件仓库名随意，一个用来放博客静态页面这个仓库名必须是github用户名.github.io

创建成功

在本地生成一个key

所在位置C:\Users\用户名\.ssh

在博客源文件的仓库设置Settings->Secrets里添加刚刚生成的私钥github-actions-deploy，名称为ACTION_DEPLOY_KEY

在博客Github Pages静态文件仓库Settings->Deploy keys添加刚生成的公钥，名称随意，但是要勾选Allow write access

添加Github Action
自动构建

设置自己的域名

如果我们自己买了域名就可以用自己的域名去访问博客

比如在阿里云中买的域名，只要在阿里云中使用CNAME方式解析指向github用户名.github.io

在github中将自己的域名设置上去即可

可能出现的问题：每次提交都要重新设置GitHub Pages绑定的域名

在根目录下的source目录中添加一个CNAME文件，全部大写并且没有后缀

里面的内容就是你的域名xxx.com

注意

如果在博客中配置了主题，请注释掉.gitignore的一些配置

主题或者其他插件通过Git clone安装的一些文件中可能会有.git文件导致无法上传到仓库，手动删除即可
