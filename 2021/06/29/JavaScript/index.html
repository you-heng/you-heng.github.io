递归

什么是递归

函数的内部语句可以调用这个函数自身，从而发起对函数的一次迭代，在新的迭代中，又会执行调用函数自身的语句

从而又产生一次迭代。当函数执行到某一处时，不再进行新的迭代，函数被一层层的返回，函数被递归

递归的要素：

边界条件，确定递归何时终止，也称为递归出口

递归模式，大问题时如果分解为小问题，也称为递归体

闭包

JavaScript 中函数会产生闭包。闭包时函数本身和该函数声明时所处的环境状态的组合

函数能够记忆住其定义时所处的环境，即使函数不在其定义的环境中被调用，也能访问定义时所处环境的变量

JavaScript 中，每次创建函数时都会创建闭包。闭包的特性需要将函数换一个地方执行才能被观察出来

闭包很有用，因为它允许我们将数据与操作该数据的函数关联起来。这与面向对象编程有少许相似之处

闭包的作用：记忆性，模拟私有变量

当闭包产生时，函数所处环境的状态会始终保持在内存中，不会再外层函数条用后被自动清除。这就是闭包的记忆性

闭包记忆性

创建体温检测函数，可以检查体温 n 是否正常，函数会返回布尔值

闭包模拟私有变量

定义一个变量，要求这个变量只能进行加乘操作，不能进行其他操作

在 JavaScript 中没有私有属性的概念，只能用闭包来模拟

不能滥用闭包，否则会造成网页的性能问题，严重时可能导致内存泄露。所谓内存泄漏是指程序中已动态分配的内存由于某种原因为释放或无法释放

事件委托

批量添加事件监听会带来性能问题，会导致监听数量太多，内存消耗非常大，单个的事件处理函数都是不同的函数，这些函数本身也会占用内存
  • 批量添加事件监听：ul 标签下有一批 li 标签，点击哪个 li 就改变它的颜色
  • 新增元素动态绑定事件。ul 标签下有 li 元素，添加一个按钮，点击按钮就添加一个 li 元素，并且每个添加的 li 元素也要有事件监听，实现点击哪个 li 哪个 li 就变红

  • 动态绑定事件的问题

    新增元素必须分别添加事件监听，不能自动获得事件监听

    大量事件监听、大量事件处理函数都会产生大量消耗内存

    事件冒泡：利用事件冒泡机制，将后台元素事件委托给祖先元素。给 ul 绑定监听事件，点击 ul 下的任何一个 li 都会通过事件冒泡传给祖先元素

    事件委托使用场景：大量类似元素需要批量添加事件监听时，使用事件委托可以减少内存开销。动态元素节点上树时，事件委托可以让新上树的元素具有事件监听

    使用事件委托时需要注意的事项：onmouseenter 和 onmouseover 都表示鼠标进入，他们的区别时 onmouseenter 不冒泡，onmouseover 冒泡

    使用事件委托时要注意：不能委托不冒泡的事件给祖先元素

    最内层元素不能有额外的元素了

    事件委托通常需要结合 e.target 属性

用 new 调用函数的四步走

new 函数()new 操作符和面向对象息息相关

js 规定，使用 new 操作符调用函数会进行四步走

  1. 函数体内会自动创建出一个空白对象

  2. 函数的上下文(this)会指向这个对象

  3. 函数体内的语句会执行

  4. 函数会自动返回上下文对象，即使函数没有 return 语句

构造函数的 prototype 属性是它实例的原型

JavaScript 规定：实例可以打点访问它的原型的属性和方法，这被称为原型链查找

  • hasOwnProperty

hasOwnProperty 方法可以检查对象是否真正自己拥有某种属性或者方法

  • in

in 运算符只能检查某个属性或方法是否可以被对象访问，不能检查是否是自己的属性或方法

在 prototype 上添加方法

把方法直接添加到实例身上的缺点：每个实例和每个实例的方法函数都是内存中不同的函数，造成了内存的浪费。解决办法：将方法写道 prototype 上

把方法写到原型上并不会造成紊乱，还可以节省内存

原型链的终点

Object.prototype.__proto__ 答案是 null 是原型的重点

继承

继承描述了两个类之间的is a kind of关系，比如学生一种人，所有人类和学生之间就构成了继承关系

父类是超类或者基类，子类是派生类

子类丰富了父类，让类描述得更具体、更细化

实现继承的关键在于：子类必须拥有父类的全部属性和方法，同时子类还应该能定义自己特有的属性和方法

使用 JavaScript 特有的原型链特性来实现继承，是普遍的做法

面向对象的本质：定义不同的类，让类的实例工作

面向对象的优点：程序编写更清晰，代码结构更严密、使代码更健壮更利于维护

面向对象经常用到的场合：需要封装和复用性的场合(组件思维)

正则表达式

正则表达式描述了字符串的构成模式，经常被用于检查字符串是否符合预定的格式要求

正则表达式按位描述规则，是指它是一位一位的描述字符串的构成形式

比如检查字符串是不是这样的，以字母 m 开头，然后是 3 个数字，最后以字母 n 结尾/^m\d\d\dn$/

创建正则表达式

使用/内容/的语法形式，可以快速创建正则表达式

也可以使用new RegExp('内容')的形式，创建正则表达式

使用 typeof 运算符检查正则表达式的类型，结果是 object

元字符

元字符是指一位指定类型的字符

注意：如果使用new RegExp()写法，反斜杠需要多写一个，因为反斜杠是转义的意思，所以需要把它转义回来

在特殊字符之前的反斜杠表示下一个字符不是特殊字符，应该按照字面意思理解/^\.$/匹配字符串是否是一个点

不管一个符号有没有特殊意义，都可以在其之前加上一个\以确保它表达的是这个符号本身

元字符 功能
\d 匹配一个数字[0~9]
\D 匹配一个非数字字符[^0~9]
\w 匹配一个单字字符(字母，数字或者下划线)[A-Za-z0-9_]
\W 匹配一个非单字字符[^a-za-z0-9_]
\s 匹配一个空白字符，包括空格，制表符和换行符
. 任意字符
^ 任意开头
$ 任意结尾
  • 方括号表示法

使用方括号，比如[xyz]，可以创建一个字符集合，表示匹配方括号中的任意字符

量词

量词 意义
* 匹配前一个表达式 0 次或多次，等价于{0,}
+ 匹配前面一个表达式 1 次或者多次。等价于{1,}
? 匹配前面一个表达式 0 次或者 1 次，等价于{0,1}
{n} n 是一个正整数，匹配了前面一个字符刚好出现了 n 次
{n,} n 是一个正整数，匹配前一个字符至少出现了 n 次
{n,m} n 和 m 都是整数，匹配前面的字符至少 n 次，最多 m 次
